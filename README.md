## How I approach the work

- **People first** — useful software starts with understanding the person using it.
- **Clarity over cleverness** — code should be easy to read, reason about, and maintain.
- **Ship, learn, improve** — momentum and feedback beat waiting for imaginary perfection.
- **Own the details** — performance, accessibility, and polish are part of the product.
